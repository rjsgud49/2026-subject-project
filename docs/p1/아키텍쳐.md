# 아키텍처 (P1)

## 1. 개요

- P1 단계의 시스템 구성과 배포 관점을 단순하게 정의합니다.
- 확장(캐시, 큐, 결제 등)은 P2·P3에서 추가합니다.

---

## 2. 시스템 구성도 (P1)

```
[클라이언트 브라우저]
        │
        │ HTTP/HTTPS
        ▼
[웹 서버 / SPA 호스팅]
        │
        ├── 정적 자산 (HTML, JS, CSS)
        │
        │ API 요청
        ▼
[백엔드 API 서버]
        │
        ├── REST API (OpenAPI 스펙)
        ├── 비즈니스 로직
        │
        │ SQL
        ▼
[데이터베이스 (RDB)]
```

- **영상 파일**: P1에서는 URL로 외부 스토리지 또는 단일 서버 경로 제공. 스트리밍은 해당 URL을 플레이어에 넘겨 재생.

---

## 3. 레이어 (백엔드)

- **Controller**: HTTP 요청/응답, 파라미터 검증
- **Service**: 비즈니스 로직 (수강신청, 진도 저장, Q&A 등)
- **Repository (또는 DAO)**: DB 접근
- **Model/Entity**: ERD 기준 엔티티

---

## 4. 배포 (가이드)

- **로컬**: 백엔드·DB 로컬 실행, 프론트는 로컬 또는 동일 서버에서 정적 파일 서빙
- **단일 서버**: 백엔드 + DB 한 서버에 배포해도 P1에서는 충분
- **환경 변수**: DB URL, 시크릿 등은 코드에 직접 넣지 않고 환경 변수로 관리

---

## 5. P2·P3에서 확장 예

- **인증**: JWT, 세션, OAuth
- **역할**: 관리자/강사/학생별 API·메뉴 제어
- **결제**: PG 연동, 웹훅
- **모니터링·로깅**: APM, 로그 수집
- **캐시**: Redis (목록·상세 캐싱)
- **파일/영상**: Object Storage, CDN
