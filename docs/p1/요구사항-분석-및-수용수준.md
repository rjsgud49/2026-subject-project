# 요구사항 분석 및 수용 수준 (P1)

## 1. 개요

- **대상**: 면접 인강 플랫폼 P1 (고객·학생 기능)
- **목적**: 기능/비기능 요구사항을 정리하고, P1에서 수용(구현)할 수준을 명시

| NF-04 | 보안 | 로그인 없이도 목록/상세 조회 가능(선택), 수강/질문은 사용자 식별 필수 |

---

## 4. 수용 수준 요약

- **필수(Must)**: P1에서 반드시 구현하는 기능
- **선택(Should)**: 시간이 허락하면 P1에서 구현, 아니면 P2/P3로 이관 가능
- **미수용(Will not in P1)**: 실 결제, 역할 분리, 모니터링·알림 등은 P2·P3에서 처리

위 표의 ✅는 P1 수용, ⭕는 선택 수용으로 이해하면 됨.
