# 서브에이전트 구현 및 체크리스트 (P1)

## 1. 목적

- P1 범위의 기능을 구현한 뒤, 스스로 또는 서브에이전트로 검증할 때 사용하는 체크리스트입니다.

---

## 2. 검색

- [ ] GET /courses — 키워드(q) 검색 시 해당 강의 목록 반환
- [ ] GET /courses — category, difficulty, min_price, max_price 필터 적용
- [ ] GET /courses — sort, page, size 적용 후 올바른 목록·총 개수 반환
- [ ] GET /courses — 공개(is_published) 강의만 노출
- [ ] 프론트: 검색창·필터·정렬 UI 동작, 결과 카드 클릭 시 상세 이동

---

## 3. 수강신청

- [ ] GET /courses/{id} — 강의 상세 + 섹션·영상 목록 반환
- [ ] POST /cart — 장바구니 담기, 이미 담긴 강의/이미 수강 중이면 409
- [ ] GET /cart — 내 장바구니 목록
- [ ] DELETE /cart/{courseId} — 장바구니에서 제거
- [ ] POST /enrollments — 수강신청(간이 결제 후), 중복 시 409
- [ ] 프론트: 상세 → 장바구니 담기, 장바구니 페이지, 결제하기 → 내 수업 이동

---

## 4. 내 수업 조회

- [ ] GET /enrollments — 내 수강 목록, 진도율·수료 여부 포함
- [ ] GET /enrollments?status=active|completed — 필터 동작
- [ ] GET /enrollments/{id} — 이어보기용 마지막 영상·위치 반환
- [ ] 프론트: 내 수업 목록, 진도율 표시, 이어보기 버튼 동작

---

## 5. 수업 듣기·진도

- [ ] GET /enrollments/{id}/progress — 영상별 진도 목록
- [ ] PUT /enrollments/{id}/videos/{videoId}/progress — last_second, completed 저장
- [ ] 진도율 계산: 완료 영상 수 / 전체 영상 수
- [ ] 수료: 모든 영상 완료 시 enrollment.status = completed
- [ ] 프론트: 영상 재생, 목차 클릭 시 해당 영상 로드, 재생 속도, 진도 자동 저장(주기적 호출)

---

## 6. 질문하기 (Q&A)

- [ ] GET /courses/{id}/questions — 강의별 질문 목록, 페이징
- [ ] POST /courses/{id}/questions — 질문 등록 (title, body)
- [ ] GET /questions/{id} — 질문 상세 + 답변 목록
- [ ] PUT /questions/{id} — 질문 수정 (본인만)
- [ ] DELETE /questions/{id} — 질문 삭제 (본인만)
- [ ] POST /questions/{id}/answers — 답변 등록
- [ ] 프론트: Q&A 목록, 질문 작성/상세, 답변 목록 표시

---

## 8. 서브에이전트 활용

- 위 체크리스트 항목별로 “해당 API 호출 → 기대 응답 확인” 시나리오를 만들어 자동/반자동 검증 가능
- 프론트는 “해당 페이지 접속 → 버튼 클릭 → URL/상태 변경 확인” 시나리오로 보완
