# P1 프로젝트에서 데이터베이스(PostgreSQL)를 쓰는 이유

## 1. 한 줄 요약

**강의·수강·장바구니·진도·Q&A처럼 “사라지면 안 되는 정보”를 안전하게 저장하고, 여러 API가 같은 데이터를 일관되게 읽고 쓰기 위해** 관계형 DB(PostgreSQL)를 사용합니다.

---

## 2. 무엇을 위해 쓰는가

| 저장 대상 | DB를 쓰는 이유 |
|-----------|----------------|
| **사용자** | 수강·장바구니·질문 작성자를 특정하려면 사용자 정보가 DB에 있어야 합니다. |
| **강의·섹션·영상** | 목록/상세 검색, 커리큘럼, 재생 URL 등 구조화된 데이터를 테이블로 관리합니다. |
| **장바구니** | 새로고침·재접속 후에도 담긴 강의가 유지되도록 서버(DB)에 저장합니다. |
| **수강신청(등록)** | 어떤 사용자가 어떤 강의를 샀는지 기록해 “내 강의실”에만 보이게 합니다. |
| **시청 진도** | 영상별 시청 위치·완료 여부를 저장해 **이어보기**·**진도율**·**수료** 판단에 씁니다. |
| **Q&A** | 질문·답변이 누적되고, 수정/삭제 시 “본인만” 같은 규칙을 user_id와 함께 적용합니다. |

즉, **브라우저를 꺼도 유지되어야 하는 모든 비즈니스 데이터**가 DB에 있습니다.

---

## 3. 왜 파일·메모리만으로는 부족한가

- **메모리만 사용**: 서버 재시작 시 데이터가 사라짐.  
- **JSON 파일만 사용**: 동시에 여러 요청이 쓸 때 깨지기 쉽고, 검색·조인·트랜잭션이 어렵습니다.  
- **RDB(PostgreSQL)**:  
  - **트랜잭션**으로 “장바구니 비우기 + 수강 등록”을 한 번에 안전하게 처리  
  - **유일 제약**(같은 강의 중복 수강 방지 등)  
  - **인덱스**로 강의 목록·Q&A 조회 속도 확보  

요구사항명세서(NFR-D01~D04)에서도 RDB 기반 영속 저장과 무결성을 전제로 하고 있습니다.

---

## 4. 정리

데이터베이스는 **인강 플랫폼의 “기억”** 역할을 합니다.  
프론트엔드·REST API는 그 위에서 조회·가입·수강·진도·질문을 처리하고, PostgreSQL이 그 상태를 **공유된 진실(source of truth)** 로 유지합니다.
