# 개발계획서 - 면접 인강 플랫폼 P1

## 1. 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 면접 인강 플랫폼 P1 |
| **범위** | 고객(학생) 기능: 검색, 수강신청, 내 수업, 수업 듣기, Q&A |
| **목표** | 백엔드 API, 프론트엔드 웹, DB 설계·구현 완료 및 인수 기준 충족 |

---

## 2. 개발 단계 및 일정

### 2.1 단계 구분

| 단계 | 명칭 | 주요 활동 | 산출물 |
|------|------|------------|--------|
| 1 | 분석·설계 | 요구사항 확정, ERD·API 설계, UI 와이어프레임 | 기획서, 요구사항명세서, ERD, api.yml, UI 개요 |
| 2 | 백엔드 구현 | API 서버, DB 스키마·마이그레이션, 비즈니스 로직 | REST API, DB |
| 3 | 프론트엔드 구현 | 화면 개발, API 연동, 영상 플레이어·진도 연동 | 웹 UI |
| 4 | 연동·테스트 | 통합 테스트, 인수 조건 검증, 버그 수정 | 테스트 결과, 수정 반영 |
| 5 | 제출·정리 | 문서 정리, 제출 폴더 구성, 최종 점검 | submit 폴더 산출물 |

### 2.2 일정(WBS) 가이드

| 주차(예시) | 단계 | 주요 작업 |
|------------|------|------------|
| 1주 | 분석·설계 | 기획서·요구사항명세서·ERD·API 스펙 확정, 개발계획서 작성 |
| 2주 | 백엔드 | DB 생성, 강의·장바구니·수강·진도·Q&A API 구현 |
| 3주 | 프론트엔드 | 검색·상세·장바구니·내 수업·수업 듣기·Q&A 화면 구현 |
| 4주 | 연동·테스트 | API–프론트 연동, 진도 저장·이어보기 검증, 인수 체크리스트 수행 |
| 5주(버퍼) | 제출·정리 | 문서 보완, submit 제출용 정리 |

- 실제 일정은 과제 마감일·팀 규모에 맞게 조정한다.

## 8. 개발 스택 및 버전

### 8.1 사용 버전 요약

| 구분 | 항목 | 사용 버전 | 비고 |
|------|------|-----------|------|
| **런타임** | Node.js | **20 LTS** | `node -v` 확인, LTS 권장 |
| **백엔드** | NestJS | **10.x** | `@nestjs/core` 등 |
| **백엔드 언어** | TypeScript | **5.x** | NestJS 기본 |
| **프론트** | React | **18.x** | |
| **프론트 빌드** | Vite | **5.x** | |
| **프론트 언어** | TypeScript | **5.x** | |
| **DB** | PostgreSQL | **15** | 로컬/호스팅 동일 버전 권장 |
| **패키지 관리** | npm / pnpm | npm **10.x** 또는 pnpm **9.x** | 프로젝트 통일 |

### 8.2 백엔드 (NestJS)

| 항목 | 버전 |
|------|------|
| Node.js | 20.x LTS |
| NestJS (CLI: `@nestjs/cli`) | 10.x |
| TypeScript | 5.x |
| PostgreSQL 클라이언트/ORM | TypeORM 0.3.x 또는 Prisma 5.x (팀 선택) |

- **생성 예시**: `npm i -g @nestjs/cli@10` 후 `nest new backend`

### 8.3 프론트엔드 (React + Vite)

| 항목 | 버전 |
|------|------|
| Node.js | 20.x LTS (백엔드와 동일) |
| React | 18.x |
| Vite | 5.x |
| TypeScript | 5.x |

- **생성 예시**: `npm create vite@5 frontend -- --template react-ts`

### 8.4 데이터베이스

| 항목 | 버전 |
|------|------|
| PostgreSQL | 15 또는 16 |
| 마이그레이션 | NestJS 프로젝트에서 선택 (TypeORM migrations / Prisma migrate 등) |

### 8.5 개발 환경 권장

| 항목 | 권장 |
|------|------|
| OS | Windows / macOS / Linux |
| IDE | VS Code 등 (동일 에디터 권장) |
| DB 클라이언트 | DBeaver |
| API 테스트 | Postman,REST Client (VS Code) |

- 버전 변경 시 본 문서와 `기술스택-및-데이터베이스.md`를 함께 수정한다.
