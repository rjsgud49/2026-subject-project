# 디자인 시스템 드래프트 v2

> 이 문서는 프로젝트의 시각적 기준을 정의한다.
> "어떻게 보여야 하는가"보다 "왜 이렇게 보여야 하는가"에 집중한다.
> 디자이너와 개발자가 같은 언어로 대화하기 위한 단일 진실의 원천(Single Source of Truth)이다.

---

## 1. 디자인 철학

군더더기 없이 기능에 집중하는 UI를 목표로 한다.
화려한 그래픽보다 **명확한 정보 전달**을, 복잡한 인터랙션보다 **예측 가능한 동작**을 우선한다.

사용자가 화면을 보는 순간 "어디를 눌러야 하는지", "지금 어떤 상태인지"를 직관적으로 파악할 수 있어야 한다.
좋은 디자인은 눈에 띄지 않는다. 사용자가 목적을 달성하는 과정에서 디자인이 방해가 되지 않을 때 비로소 성공이다.

### 핵심 원칙

**일관성** — 같은 의미를 가진 요소는 항상 같은 모양과 위치에 있어야 한다. 사용자는 한번 학습한 패턴을 반복 적용한다.

**밀도 조절** — 콘텐츠가 많다고 공백을 줄이지 않는다. 오히려 콘텐츠가 많을수록 더 신중하게 여백을 배분한다. 숨을 쉴 공간 없는 UI는 피로하다.

**계층 명확화** — 모든 페이지에는 시선을 끄는 1순위 요소가 반드시 하나 존재해야 한다. 중요한 것은 크고 굵게, 덜 중요한 것은 작고 흐리게.

**피드백** — 모든 인터랙션에는 시각적·체감적 응답이 따른다. 클릭했는데 아무것도 바뀌지 않는 UI는 고장난 것처럼 느껴진다.

**정직함** — 로딩 중이면 로딩 중이라고, 오류가 났으면 무엇이 잘못됐는지 명확히 알려준다. 사용자를 무한 대기시키지 않는다.

## 2. 디자인 토큰

디자인 토큰은 디자인 결정을 코드로 표현한 것이다.
색, 크기, 간격 같은 값을 하드코딩하지 않고 토큰으로 관리하면 전체 시스템을 일관되게 유지할 수 있다.

```css
:root {
  /* Primary */
  --color-primary-50:  #EFF6FF;
  --color-primary-500: #3B82F6;
  --color-primary-600: #2563EB;
  --color-primary-700: #1D4ED8;

  /* Neutral */
  --color-neutral-50:  #FAFAFA;
  --color-neutral-100: #F4F4F5;
  --color-neutral-200: #E4E4E7;
  --color-neutral-400: #A1A1AA;
  --color-neutral-500: #71717A;
  --color-neutral-600: #52525B;
  --color-neutral-700: #3F3F46;
  --color-neutral-800: #27272A;
  --color-neutral-900: #18181B;

  /* Radius */
  --radius-sm:  4px;
  --radius-md:  8px;
  --radius-lg:  12px;
  --radius-xl:  16px;
  --radius-full: 9999px;

  /* Shadow */
  --shadow-xs: 0 1px 2px 0 rgba(0,0,0,0.05);
  --shadow-sm: 0 1px 3px 0 rgba(0,0,0,0.1), 0 1px 2px -1px rgba(0,0,0,0.1);
  --shadow-md: 0 4px 6px -1px rgba(0,0,0,0.1), 0 2px 4px -2px rgba(0,0,0,0.1);
  --shadow-lg: 0 10px 15px -3px rgba(0,0,0,0.1), 0 4px 6px -4px rgba(0,0,0,0.1);
  --shadow-xl: 0 20px 25px -5px rgba(0,0,0,0.1), 0 8px 10px -6px rgba(0,0,0,0.1);

  /* Duration */
  --duration-fast:   100ms;
  --duration-normal: 200ms;
  --duration-slow:   300ms;
}
```

## 3. 컬러 시스템

색은 적게 쓸수록 강하다. 전체 UI에서 사용하는 색상 종류를 의도적으로 제한한다.
Primary 1가지, Neutral 1가지, Semantic 4가지로 구성한다.

색상을 추가하고 싶을 때는 "기존 색으로 표현이 불가능한가"를 먼저 질문한다.

---

### 3-1. Primary 컬러

주요 액션, CTA 버튼, 링크, 포커스 상태, 선택된 상태에 사용한다.
브랜드를 대표하는 색이므로 함부로 교체하지 않는다.

```
primary-50:  #EFF6FF   배경 하이라이트, 선택 항목 배경
primary-100: #DBEAFE   chip, badge 배경
primary-200: #BFDBFE   비활성 primary 요소, 프로그레스 배경
primary-300: #93C5FD
primary-400: #60A5FA
primary-500: #3B82F6   기본값 — 가장 많이 사용
primary-600: #2563EB   hover 상태
primary-700: #1D4ED8   active / pressed 상태
primary-800: #1E40AF
primary-900: #1E3A8A   매우 강조가 필요한 경우
```

**사용 기준**
- 버튼, 링크, 체크박스, 토글, 탭 활성화: `primary-500`
- 버튼 hover: `primary-600` / active: `primary-700`
- 선택된 항목 배경: `primary-50`, 텍스트: `primary-700`
- 포커스 링: `ring-2 ring-primary-500 ring-offset-2`
- 본문 텍스트에 primary 색 사용 금지 — 링크에만 허용

---

### 3-2. Neutral 컬러

UI의 90%를 차지한다. 텍스트, 배경, 테두리, 구분선 전부 이 계열에서 가져온다.
Zinc 계열 기준 (Gray보다 차갑고 현대적이다).

```
neutral-0:   #FFFFFF   카드, 모달 배경
neutral-50:  #FAFAFA   페이지 배경
neutral-100: #F4F4F5   섹션 구분, 입력 비활성 배경
neutral-200: #E4E4E7   구분선, 테두리 기본
neutral-300: #D4D4D8   비활성 테두리, hover 테두리
neutral-400: #A1A1AA   placeholder, 비활성 텍스트
neutral-500: #71717A   보조 텍스트 (날짜, 설명, 캡션)
neutral-600: #52525B   중간 계층 텍스트
neutral-700: #3F3F46   서브 헤딩, 강조 보조 텍스트
neutral-800: #27272A   본문 텍스트 기본값
neutral-900: #18181B   헤딩, 강조 텍스트, 다크 배경
```

---

### 3-3. Semantic 컬러

기능적 의미가 있는 색이다. 상황에 맞게만 사용하고 장식 목적으로 남용하지 않는다.

**Success — 완료, 확인, 정상**
```
success-50:  #F0FDF4
success-100: #DCFCE7
success-500: #22C55E
success-600: #16A34A   텍스트, 아이콘
success-700: #15803D   강조
```

**Warning — 주의, 확인 필요**
```
warning-50:  #FFFBEB
warning-100: #FEF3C7
warning-500: #F59E0B
warning-600: #D97706
warning-700: #B45309
```

**Error — 오류, 실패, 위험**
```
error-50:  #FFF1F2
error-100: #FFE4E6
error-500: #EF4444
error-600: #DC2626
error-700: #B91C1C
```

**Info — 정보, 안내**
```
info-50:  #EFF6FF
info-500: #3B82F6   (primary와 동일 — 별도 토큰 불필요)
info-600: #2563EB
```

**Semantic 색 사용 기준**
- 배경: 50 계열 (매우 연하게, 강조가 아닌 구분용)
- 아이콘, 텍스트: 600~700 계열 (WCAG AA 충족)
- 테두리: 200~300 계열
- 500 계열 — 아이콘 단독 사용만 허용, 텍스트에는 600 이상

---

### 3-4. 다크 모드

다크 모드는 단순히 색을 반전하지 않는다.
밝은 배경에서 그림자가 높이를 나타냈다면, 어두운 배경에서는 **배경색 밝기 차이(elevation)**로 대체한다.

```
레벨 0 (페이지 배경):   #09090B  neutral-950
레벨 1 (카드):          #18181B  neutral-900
레벨 2 (팝오버/드롭다운): #27272A  neutral-800
레벨 3 (모달):          #2D2D30
구분선:                 #3F3F46  neutral-700
본문 텍스트:            #E4E4E7  neutral-200
보조 텍스트:            #71717A  neutral-500
비활성 텍스트:          #52525B  neutral-600
```

**다크 모드 원칙**
- 흰색(`#FFFFFF`) 텍스트를 배경에 직접 쓰지 않는다. `neutral-200`이 눈에 덜 피로하다
- primary 색은 그대로 유지 (파란색 계열은 어두운 배경에서도 충분히 보인다)
- 그림자 대신 `ring-1 ring-white/10` 같은 테두리로 카드를 구분한다

## 4. 타이포그래피

### 4-1. 폰트 패밀리

**한글 / 기본 UI**: Pretendard Variable
**영문 / 숫자**: 시스템 폰트 스택 (Inter 계열과 자연스럽게 혼합)
**코드**: JetBrains Mono (fallback: Fira Code, Consolas, monospace)

```css
/* 기본 UI 폰트 */
font-family: 'Pretendard Variable', 'Pretendard',
  -apple-system, BlinkMacSystemFont,
  'Apple SD Gothic Neo', 'Noto Sans KR',
  sans-serif;

/* 코드 폰트 */
font-family: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;
```

Pretendard를 쓰는 이유: 웨이트별 굵기 차이가 뚜렷하고, 한·영 혼용 시 자간이 자연스러우며, Variable 폰트이므로 단일 파일로 모든 웨이트를 처리한다.

---

### 4-2. 타입 스케일

4px 그리드에서 파생된 스케일이다. 이 스케일 밖의 임의 크기를 사용하지 않는다.

```
스케일    크기   line-height   letter-spacing   용도
──────────────────────────────────────────────────────────
xs        12px   1.5           +0.02em          캡션, 태그, 보조 레이블
sm        14px   1.5           +0.01em          보조 텍스트, 메타 정보
base      16px   1.6           0                본문 기본
lg        18px   1.6           -0.01em          서브 타이틀, 강조 본문
xl        20px   1.5           -0.01em          카드 헤딩
2xl       24px   1.4           -0.02em          섹션 타이틀
3xl       30px   1.3           -0.02em          페이지 서브 타이틀
4xl       36px   1.25          -0.03em          페이지 타이틀
5xl       48px   1.2           -0.03em          히어로 헤딩
6xl       60px   1.1           -0.04em          랜딩 대형 카피
```

---

### 4-3. 폰트 웨이트

```
400 Regular   본문 텍스트, 설명, placeholder
500 Medium    인풋 라벨, 네비게이션 링크, 버튼 내 텍스트
600 SemiBold  카드 타이틀, 강조 텍스트, 테이블 헤더
700 Bold      페이지 타이틀, 섹션 헤딩, CTA 버튼
800 ExtraBold 브랜드 로고, 히어로 카피
```

한 페이지에서 3가지 이상 웨이트를 동시에 쓰지 않는다.
예외: 숫자 데이터를 강조하는 대시보드는 Bold/Regular 조합만으로 충분하다.

---

### 4-4. 텍스트 계층

```
역할             크기   웨이트       색
─────────────────────────────────────────────────────
Page Title (H1)  4xl   700 Bold     neutral-900
Section Title    2xl   700 Bold     neutral-900
Card Title       xl    600 Semi     neutral-800
Sub Title        lg    600 Semi     neutral-700
Body             base  400 Regular  neutral-800
Body Small       sm    400 Regular  neutral-600
Caption          xs    400 Regular  neutral-500
Label            sm    500 Medium   neutral-700
Code Inline      sm    400 Regular  neutral-700 + bg neutral-100
```

---

### 4-5. 줄 길이 (Line Length)

본문 가독성이 가장 좋은 줄 길이: 한글 30~50자, 영문 45~75자.

- 본문 텍스트 컨테이너 `max-width`: **680px**
- 전체 레이아웃 너비와 본문 읽기 너비는 독립적으로 관리한다
- 좁은 화면에서 강제로 2줄로 줄이는 것보다 폰트 크기를 조정하는 것이 낫다

## 5. 간격 시스템 (Spacing)

4px 기반 단일 스케일. Tailwind 기본 spacing과 동일하게 유지한다.

```
t-0.5:  2px    미세 조정 (아이콘 수직 정렬 등)
t-1:    4px    인접 요소 밀착 간격 (아이콘↔텍스트)
t-1.5:  6px
t-2:    8px    chip, badge 내부 패딩
t-2.5:  10px
t-3:    12px   인풋 내부 패딩 (세로)
t-4:    16px   컴포넌트 기본 패딩, 그룹 내 요소 간격
t-5:    20px
t-6:    24px   카드 내부 패딩
t-8:    32px   섹션 내 요소 간격
t-10:   40px
t-12:   48px   섹션 간 구분
t-16:   64px   대형 섹션 구분
t-20:   80px
t-24:   96px   히어로, 랜딩 섹션
```

### 간격 사용 기준

**컴포넌트 내부 패딩**
```
소형 (badge, chip, sm button): px-2 py-0.5 ~ px-3 py-1
중형 (button md, input):       px-4 py-2.5
대형 (card, panel):            px-6 py-5 ~ px-6 py-6
페이지 컨테이너:                px-4 md:px-6 lg:px-8
```

**요소 간 수직 간격**
```
label ↔ input:              gap-1.5 (6px)
input ↔ hint/error:         mt-1.5 (6px)
폼 필드 간:                  space-y-4 (16px)
섹션 내 카드 그리드:          gap-4 md:gap-6
섹션 간:                    space-y-12 md:space-y-16
페이지 상단 패딩:             py-8 md:py-12
```

**규칙**: 3px, 5px, 7px, 9px 같은 홀수 값과 Tailwind에 없는 임의 값(`p-[13px]`)은 사용하지 않는다.

## 6. 테두리 반경 (Border Radius)

```
none:   0px      테이블, 날카로운 구분선
sm:     4px      인라인 코드, 작은 내부 요소
base:   8px      인풋, 버튼 기본 (rounded-lg)
md:     12px     카드, 패널 (rounded-xl)
lg:     16px     대형 카드, 시트
xl:     20px     특별히 부드러운 느낌이 필요한 경우
2xl:    24px     모달 (rounded-2xl)
full:   9999px   pill 버튼, badge, avatar, progress bar
```

**사용 원칙**
- 같은 계층의 요소는 같은 radius (버튼끼리 통일)
- 내부 요소의 radius는 외부 컨테이너보다 2~4px 작게
- 카드 내부 이미지: 카드 radius - 4px
- 모달 내부 구분 영역: `rounded-lg`
- 전체 페이지에서 radius 값을 2~3종류 이상 섞지 않는다

## 7. 그림자 / Elevation

밝은 테마에서 그림자는 깊이를 나타낸다. 과하게 쓰면 어수선해 보인다.

```
shadow-xs:  0 1px 2px 0 rgba(0,0,0,0.05)
            → 버튼, 인풋 (기본 상태), 아주 미세한 부각

shadow-sm:  0 1px 3px 0 rgba(0,0,0,0.10),
            0 1px 2px -1px rgba(0,0,0,0.10)
            → 카드 기본, 드롭다운 기본

shadow-md:  0 4px 6px -1px rgba(0,0,0,0.10),
            0 2px 4px -2px rgba(0,0,0,0.10)
            → 카드 hover, 팝오버, 플로팅 버튼

shadow-lg:  0 10px 15px -3px rgba(0,0,0,0.10),
            0 4px 6px -4px rgba(0,0,0,0.10)
            → 드롭다운, 컨텍스트 메뉴, 사이드 시트

shadow-xl:  0 20px 25px -5px rgba(0,0,0,0.10),
            0 8px 10px -6px rgba(0,0,0,0.10)
            → 모달, 알림 패널, 커맨드 팔레트
```

**사용 원칙**
- 카드 기본 `shadow-sm`, hover 시 `shadow-md` + `translateY(-2px)` 조합
- 드롭다운/팝오버: `shadow-lg`
- 모달: `shadow-xl` + `backdrop-blur-sm` 오버레이
- 다크 모드: 그림자 대신 `ring-1 ring-white/[0.08]`로 elevation 표현

## 8. 컴포넌트 설계 기준

### 8-1. 버튼 (Button)

버튼의 종류는 4가지로 제한한다.

**Primary**
```
배경: primary-500 / 텍스트: white / shadow: shadow-xs
hover: primary-600 / active: primary-700 / scale-[0.98]
disabled: primary-300 opacity-60
용도: 페이지당 1개의 주요 CTA에만 사용
```

**Secondary (Outlined)**
```
배경: white / 테두리: neutral-200 / 텍스트: neutral-700 / shadow: shadow-xs
hover: bg-neutral-50 border-neutral-300
용도: Primary 옆 보조 액션, 선택지가 여러 개일 때
```

**Ghost (Text)**
```
배경: 없음 / 텍스트: neutral-600
hover: bg-neutral-100 / active: bg-neutral-200
용도: 취소, 닫기, 더보기 같은 낮은 우선순위 액션
```

**Destructive**
```
배경: error-600 / 텍스트: white
hover: error-700 / active: error-800
용도: 삭제, 계정 탈퇴 등 되돌릴 수 없는 액션에만 사용
반드시 확인 다이얼로그 앞에 배치
```

**버튼 사이즈**
```
xs:  h-7  px-2.5  text-xs   rounded-md  gap-1
sm:  h-8  px-3    text-sm   rounded-md  gap-1.5
md:  h-10 px-4    text-sm   rounded-lg  gap-2   ← 기본값
lg:  h-12 px-6    text-base rounded-lg  gap-2
xl:  h-14 px-8    text-base rounded-xl  gap-3
```

**공통 규칙**
- 로딩 상태: 스피너 + "처리 중..." 텍스트, disabled 처리 필수
- 아이콘 포함 시 아이콘 크기는 텍스트보다 1~2px 크게 (시각 보정)
- 버튼 텍스트는 동사+목적어 구조 (`저장하기` `계정 삭제` `로그인`)
- 아이콘만 있는 버튼: `aria-label` 필수, 툴팁 권장
- Primary 버튼 2개를 같은 영역에 동시 노출 금지

### 8-2. 인풋 (Input)

**기본 상태**
```
배경: white
테두리: 1px solid neutral-200
텍스트: neutral-800 / placeholder: neutral-400
radius: rounded-lg (8px)
padding: px-3 py-2.5
height: 40px (md)
font-size: text-sm
transition: border-color, box-shadow 150ms
```

**상태별 스타일**
```
hover:    border-neutral-300
focus:    border-primary-500  ring-2 ring-primary-100
error:    border-error-500    ring-2 ring-error-50
success:  border-success-500  ring-2 ring-success-50 (필요한 경우만)
disabled: bg-neutral-50 text-neutral-400 cursor-not-allowed border-neutral-200
readonly: bg-neutral-50 cursor-default
```

**레이블 규칙**
- 레이블은 항상 인풋 위에. floating label 지양 (스크린 리더 혼동)
- 필수 항목: 레이블 옆 빨간 `*` (`text-error-500`)
- 선택 항목: `(선택)` 텍스트 추가 (빨간 `*` 남용 방지)
- hint 텍스트: 인풋 아래 `text-xs text-neutral-500`
- 에러 메시지: 인풋 아래 `text-xs text-error-600` + `role="alert"`

**인풋 크기**
```
sm:  h-8  px-2.5 text-xs  rounded-md
md:  h-10 px-3   text-sm  rounded-lg  ← 기본값
lg:  h-12 px-4   text-base rounded-lg
```

**Textarea**
```
최소 높이: 5줄 (rows="5")
resize: vertical 허용, horizontal 금지
overflow-y: auto
```

네이티브 `<select>`는 스타일 제약이 크다. 커스텀 드롭다운을 사용할 것.

트리거:  인풋과 동일한 스타일 + 우측 ChevronDown 아이콘
드롭다운: white bg / rounded-xl / shadow-lg / py-1
옵션:    px-3 py-2 text-sm / hover: bg-neutral-50 / selected: bg-primary-50 text-primary-700
검색:    10개 이상 옵션이면 검색 인풋 상단에 추가
```

### 8-4. 카드 (Card)

카드는 연관된 정보를 하나의 단위로 묶는 컨테이너다.
3가지 스타일만 허용한다.

**Elevated — 기본**
```
배경: white / shadow: shadow-sm / radius: rounded-xl / padding: p-5~p-6
hover: shadow-md translateY(-2px) transition-all duration-200
```

**Outlined**
```
배경: white / 테두리: 1px solid neutral-200 / shadow: 없음 / radius: rounded-xl
hover: border-neutral-300
```

**Filled**
```
배경: neutral-50 또는 primary-50 / 테두리: 없음 / shadow: 없음 / radius: rounded-xl
```

**카드 내부 구성 순서**
1. 상단 이미지 또는 아이콘 (선택)
2. 메타 정보 (카테고리, 날짜 — `text-xs text-neutral-500`)
3. 타이틀 (`text-base font-semibold text-neutral-800`, `line-clamp-2`)
4. 설명 (`text-sm text-neutral-500`, `line-clamp-3`)
5. 통계/수치 (별점, 수강생 수 등)
6. 액션 버튼 (항상 맨 아래, `mt-auto`)

**카드 그리드**
```
기본:    grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-5
넓은형:  grid-cols-1 md:grid-cols-2 gap-6
목록형:  flex flex-col gap-3
```

sm: text-xs px-2   py-0.5 rounded-full  ← 테이블, 카드 메타
md: text-sm px-2.5 py-1   rounded-full  ← 기본값
```

**색상 조합**
```
neutral:  bg-neutral-100  text-neutral-700
primary:  bg-primary-50   text-primary-700
success:  bg-success-50   text-success-700
warning:  bg-warning-50   text-warning-700
error:    bg-error-50     text-error-700
```

**규칙**
- 뱃지 텍스트는 2단어 이내
- 아이콘 포함 시 텍스트 왼쪽 배치, 크기 `text-xs`
- 클릭 가능한 뱃지: hover/focus 스타일 필수, cursor-pointer

### 8-6. 모달 (Modal / Dialog)

**크기**
```
sm:   max-w-sm  (384px)   확인/경고 다이얼로그
md:   max-w-md  (448px)   기본 폼 모달
lg:   max-w-lg  (512px)   복잡한 폼
xl:   max-w-xl  (576px)   미리보기, 상세 정보
2xl:  max-w-2xl (672px)   대용량 콘텐츠, 에디터
full: 모바일에서 bottom sheet로 전환
```

**레이아웃 구조**
```
오버레이:   bg-black/50 backdrop-blur-sm z-30
컨테이너:   white rounded-2xl shadow-xl
헤더:       타이틀(font-semibold) + 닫기 버튼(우상단) px-6 py-4 border-b
콘텐츠:     px-6 py-5 (스크롤 필요 시 max-h + overflow-y-auto)
푸터:       px-6 py-4 border-t flex justify-end gap-2
```

**규칙**
- 열릴 때 body 스크롤 잠금
- ESC, 오버레이 클릭으로 닫기 (파괴적 액션 모달 제외)
- 초점(focus)은 모달 내부에 트랩
- 모달 안 모달 절대 금지
- 내용이 길면 헤더·푸터 고정, 내부만 스크롤

### 8-7. 테이블 (Table)

**헤더**
```
배경:     neutral-50
텍스트:   text-xs font-semibold text-neutral-500 uppercase tracking-wide
테두리:   border-b border-neutral-200
padding:  px-4 py-3
```

**행 (Row)**
```
텍스트:   text-sm text-neutral-800
padding:  px-4 py-3.5
테두리:   border-b border-neutral-100
hover:    bg-neutral-50
클릭 가능: cursor-pointer
```

**규칙**
- 숫자 데이터: 우측 정렬, 고정폭 폰트
- 텍스트: 좌측 정렬
- 액션 열: 항상 마지막 열, 우측 정렬
- 빈 상태: 테이블 내부 중앙에 빈 상태 컴포넌트 삽입
- 체크박스 열: 첫 번째 열, 너비 고정 (`w-12`)
- 긴 텍스트: `truncate max-w-xs` + 툴팁으로 전체 내용 표시
- 정렬 가능한 열: 헤더에 정렬 아이콘 표시

**모바일 대응**
- 10컬럼 이상 테이블은 모바일에서 가로 스크롤 허용 (`overflow-x-auto`)
- 중요도 낮은 열은 `hidden md:table-cell`로 숨김
- 카드 목록 뷰로 전환하는 것도 좋은 선택

### 8-9. 툴팁 (Tooltip)

```
배경:       neutral-900 (다크)
텍스트:     text-xs text-white px-2.5 py-1.5
radius:     rounded-lg
shadow:     shadow-md
등장:       opacity 0→1, translateY 4px→0, duration-150 ease-out
최대 너비:   max-w-xs
```

**규칙**
- 아이콘 전용 버튼에는 반드시 툴팁 제공
- 잘린 텍스트(`truncate`)에는 전체 내용 툴팁 제공
- 툴팁은 클릭으로 활성화하지 않는다 — hover/focus에서만

### 8-10. 토스트 / 알림 (Toast)

```
위치:       화면 우하단 (bottom-4 right-4) 또는 상단 중앙
너비:       min-w-[280px] max-w-sm
배경:       white / shadow-xl / rounded-xl
padding:    px-4 py-3.5
아이콘:     좌측 고정 (Success/Warning/Error/Info 구분)
닫기:       우측 X 버튼 (선택)
지속시간:   3~5초 자동 닫힘 (에러는 수동 닫기)
등장:       translateX(100%)→0 또는 translateY(100%)→0
사라짐:     opacity 1→0 duration-200
```

**토스트 유형**
```
success:  체크 아이콘 + success-600 텍스트
error:    X 아이콘 + error-600 텍스트 + 수동 닫기 권장
warning:  경고 아이콘 + warning-700 텍스트
info:     정보 아이콘 + neutral-700 텍스트
```

### 8-14. 빈 상태 (Empty State)

데이터가 없을 때 빈 화면을 절대 보여주지 않는다.

**구성**
```
1. 아이콘/일러스트:    48~64px, text-neutral-300
2. 타이틀:            text-base font-semibold text-neutral-700
3. 설명:              text-sm text-neutral-500, 최대 2줄
4. CTA 버튼:          (선택) Primary 버튼
```

**텍스트 톤 가이드**
```
❌ "데이터가 없습니다."
❌ "결과 없음"
✅ "아직 등록된 강의가 없어요. 첫 강의를 만들어보세요!"
✅ "검색 결과가 없어요. 다른 키워드로 시도해보세요."
✅ "장바구니가 비어 있어요. 마음에 드는 강의를 담아보세요."
```

### 8-15. 스켈레톤 로딩 (Skeleton)

데이터 로딩 중에는 스피너보다 스켈레톤을 우선 사용한다.
스켈레톤은 실제 콘텐츠의 레이아웃과 형태를 반영해야 한다.

```css
/* 기본 */
.skeleton {
  background-color: neutral-200;
  border-radius: 4px;
  animation: pulse 2s cubic-bezier(0.4, 0, 0.6, 1) infinite;
}
```

**Tailwind 사용**
```
h-4 w-3/4 rounded bg-neutral-200 animate-pulse
h-4 w-1/2 rounded bg-neutral-200 animate-pulse mt-2
```

**규칙**
- 스켈레톤 개수는 예상 콘텐츠 개수와 일치시킨다 (3개면 3개)
- 텍스트 스켈레톤: 제목은 너비 60~75%, 설명은 너비 40~55%
- 이미지 스켈레톤: aspect-ratio를 실제와 동일하게 유지
- 1초 이상 걸리는 경우에만 스켈레톤 표시 (짧은 로딩에는 오버킬)

## 9. 폼 / 유효성 검사 UX

### 9-1. 검증 타이밍

```
실시간 검증 (onChange):   비밀번호 강도, 중복 확인
제출 후 검증 (onBlur):    이메일 형식, 필수 필드
일괄 검증 (onSubmit):     모든 필드 최종 확인
```

onBlur 타이밍이 대부분의 경우 최적이다.
사용자가 타이핑 중일 때 에러를 표시하면 방해가 된다.

### 9-2. 에러 메시지 작성 기준

```
❌ "유효하지 않은 이메일"           (기술적 용어)
❌ "필드를 입력해주세요"            (뻔함)
✅ "이메일 주소를 올바르게 입력해 주세요. (예: name@email.com)"
✅ "비밀번호는 8자 이상, 영문+숫자를 포함해야 합니다."
✅ "이미 사용 중인 이메일입니다. 로그인 페이지로 이동하시겠어요?"
```

- 에러 위치: 해당 필드 바로 아래 (전체 상단 배너 ❌)
- 에러 색: `error-600` + `role="alert"`
- 에러 해결 시 즉시 제거 (다음 제출까지 기다리지 않음)

### 9-3. 폼 레이아웃

```
단일 컬럼 폼:   max-w-md 이내 유지 (2컬럼 폼은 인지 부하 증가)
레이블 위치:    항상 인풋 위 (인라인 레이블/placeholder 대체 금지)
버튼 위치:      우측 정렬 (취소 좌측 + 확인 우측)
필수 표시:      레이블 옆 * (색: error-500, 양이 많으면 "* 필수 항목" 전체 안내)
```

## 11. 내비게이션 패턴

### 11-1. 상단 네비게이션 바 (Top Nav)

```
높이:         h-14 (56px)
배경:         white border-b border-neutral-200
고정:         sticky top-0 z-10
구성:         [로고] [메인 메뉴] [검색] [CTA 버튼] [사용자 메뉴]
로고:         좌측 고정 / 홈 링크
검색:         중앙 또는 우측 (breadcrumb 없는 경우 중앙 허용)
사용자 메뉴:  우측 / 아바타 or 이름 + 드롭다운
```

**활성 메뉴 표현**
```
하단 밑줄:    border-b-2 border-primary-500 text-primary-600
배경 강조:    bg-primary-50 text-primary-700 rounded-lg
```

### 11-2. 브레드크럼 (Breadcrumb)

```
구분자:    / 또는 ChevronRight (크기 12px, neutral-400)
현재 위치: text-neutral-500 (클릭 불가, aria-current="page")
이전 경로: text-neutral-600 hover:text-primary-600
최대 표시: 3~4단계. 이상이면 중간 생략 + "..."
```

## 12. 인터랙션 / 모션

### 12-1. Duration 기준

```
즉각 피드백 (hover, focus):        100~150ms
상태 전환 (탭, 체크박스, 토글):     150~200ms
요소 등장/사라짐 (드롭다운, 툴팁):  150~200ms
모달 열기/닫기:                     200~250ms
페이지 전환 (router fade):          200~300ms
대형 애니메이션 (진행 바 등):        300~500ms
```

600ms를 초과하는 애니메이션은 특별한 서술적 이유가 없으면 사용 금지.

### 12-2. Easing 기준

```
ease-out    (cubic-bezier 0,0,0.2,1): 요소 등장 — 처음 빠르게, 끝은 자연스럽게
ease-in     (cubic-bezier 0.4,0,1,1): 요소 사라짐 — 빠르게 사라짐
ease-in-out (cubic-bezier 0.4,0,0.2,1): 위치 이동, 크기 변화
linear:                                  스피너 같은 반복 애니메이션
spring:                                  모달, 드로어 (bounce 느낌)
```

### 12-3. 인터랙션 피드백

클릭 가능한 모든 요소는 3가지 상태를 반드시 가진다.

```
hover:   색, 배경, 그림자 변화
focus:   ring-2 ring-primary-500 ring-offset-2 (키보드 접근성)
active:  scale-[0.98] + shadow 축소 (눌리는 느낌)
```

```css
/* 버튼 active 예시 */
.btn { @apply active:scale-[0.98] active:shadow-none transition-transform duration-75; }
```

### 12-4. 페이지 전환

```
router-view에 transition 적용:
  enter: opacity 0 → 1, translateY 8px → 0, duration-200 ease-out
  leave: opacity 1 → 0, duration-150 ease-in
```

## 13. 아이콘 시스템

### 13-1. 기본 라이브러리

**Lucide** (Vue: `lucide-vue-next`) — stroke 1.5px, 정사각형 그리드 기반.
Hero Icons, Phosphor Icons도 허용하나 한 프로젝트 내 라이브러리 혼용 금지.

### 13-2. 아이콘 크기

```
xs:  12px   뱃지, chip 내부, 보조 레이블 옆
sm:  14px   테이블 셀, 인풋 내부 아이콘
md:  16px   버튼, 인풋 레이블 옆  ← 가장 많이 사용
lg:  20px   네비게이션, 리스트 아이콘
xl:  24px   강조 아이콘, 섹션 헤더
2xl: 32px   빈 상태, 카드 상단 기능 아이콘
3xl: 48px   온보딩, 랜딩 일러스트 대체
```

### 13-3. 사용 원칙

- 아이콘만으로 의미 전달 금지 — 텍스트 라벨 병행 필수 (접근성)
- 공간 제약 있을 때(좁은 툴바 등) 아이콘만 허용하되 `aria-label` + 툴팁 필수
- 아이콘 색은 인접 텍스트 색과 동일하게 유지
- 아이콘 크기는 옆 텍스트 font-size와 동일하거나 1~2px 크게 (시각 보정)
- filled(채워진) 아이콘과 outline 아이콘 혼용 금지

## 14. 접근성 (Accessibility)

WCAG 2.1 AA 기준을 최소 만족한다.

### 14-1. 색 대비

```
일반 텍스트 (< 18px):      최소 4.5:1
대형 텍스트 (≥ 18px Bold): 최소 3:1
UI 컴포넌트, 아이콘:        최소 3:1

neutral-800 on white: ~14:1  ✅
neutral-500 on white: ~7:1   ✅
neutral-400 on white: ~4.6:1 ✅ (간신히)
neutral-400 on neutral-50: ~4.2:1  ⚠ 주의
primary-600 on white: ~5.9:1 ✅
error-600   on white: ~5.8:1 ✅
```

primary-50 배경 위 텍스트는 반드시 `primary-700` 이상 사용.

### 14-2. 키보드 네비게이션

- 모든 인터랙티브 요소는 Tab으로 접근 가능해야 한다
- `outline: none` 단독 사용 절대 금지 — focus 스타일을 반드시 대체 제공
- 표준 focus ring: `ring-2 ring-primary-500 ring-offset-2`
- 모달: 열릴 때 첫 번째 인터랙티브 요소에 자동 포커스
- 모달: 닫힐 때 트리거 버튼으로 포커스 복귀
- 드롭다운: Arrow 키 네비게이션

### 14-3. ARIA 필수 적용

```
아이콘 전용 버튼:  aria-label
모달:             role="dialog" aria-modal="true" aria-labelledby
로딩 중:          aria-busy="true" aria-live="polite"
에러 메시지:       role="alert" (즉시 읽힘) 또는 aria-live="polite"
페이지 영역:       <main> <nav> <aside> <header> <footer> 랜드마크 사용
이미지:           alt 필수 (장식용은 alt="")
진행 표시:         role="progressbar" aria-valuenow aria-valuemin aria-valuemax
```

## 15. 반응형 디자인

### 15-1. 브레이크포인트

```
기본 (mobile-first): 0px~
sm:   640px   작은 태블릿, 가로 모바일
md:   768px   태블릿
lg:   1024px  작은 노트북
xl:   1280px  일반 데스크탑
2xl:  1536px  대형 모니터
```

### 15-2. 모바일 퍼스트 원칙

모든 컴포넌트는 모바일 기준으로 먼저 설계하고 점진적으로 확장한다.

```
모바일:     핵심 기능만 / 수직 레이아웃 / 탭 하단 고정
태블릿:     사이드바 등장 / 2컬럼 그리드
데스크탑:   3~4컬럼 / 전체 기능 노출
```

데스크탑에서만 동작하는 기능을 만들지 않는다.
모바일에서 숨기는 것보다 단순화하는 방향으로 설계한다.

### 15-3. 모바일 특수 패턴

```
네비게이션:  하단 탭바 (max 5개 항목) 또는 햄버거 + 드로어
모달:        bottom sheet (translateY 등장)
긴 폼:       한 화면에 3개 이상 필드 지양 → 멀티스텝
테이블:      좌우 스크롤 또는 카드 목록으로 전환
드롭다운:    full-screen 선택 시트
사이드바:    기본 숨김, 버튼으로 오버레이 드로어 열기
```

## 16. 콘텐츠 작성 지침 (UX Writing)

디자인과 텍스트는 분리할 수 없다. 버튼 라벨 하나가 전환율을 바꾼다.

### 16-1. 버튼 / CTA 텍스트

```
❌ "제출"   → ✅ "신청하기"
❌ "확인"   → ✅ "삭제하기" (파괴적 액션에서 무엇을 하는지 명시)
❌ "예"     → ✅ "네, 탈퇴할게요"
❌ "로그인" → ✅ "시작하기" (랜딩 히어로 CTA)
```

### 16-2. 에러 메시지

```
❌ "오류가 발생했습니다."           (무슨 오류인지 모름)
❌ "ERROR_CODE_412"               (기술적 코드 노출)
✅ "저장에 실패했습니다. 잠시 후 다시 시도해 주세요."
✅ "네트워크 연결을 확인하고 다시 시도해 주세요."
✅ "이미 사용 중인 이메일입니다."   (명확한 원인 + 해결 방향)
```

### 16-3. 로딩 / 처리 중 텍스트

```
단순 로딩:  "불러오는 중..."
저장 중:    "저장하는 중..."
제출 중:    "신청하는 중..."   (버튼 텍스트)
삭제 중:    "삭제하는 중..."
긴 작업:    "처리 중입니다. 잠시만 기다려 주세요."
```

### 16-4. 수치 표현

```
숫자 단위:   1,234명 / 1.2만 명 / 4.5시간 / 12분 30초
가격:        무료 / 55,000원 / 월 9,900원
날짜:        2024.03.15 / 3일 전 / 방금 전
평점:        4.8 (2,341명 평가)
진도:        42% 완료 (12/28강)
```

## 17. 컴포넌트 상태 설계 기준

모든 컴포넌트는 최소 다음 상태를 설계한다.
상태 설계 없이 컴포넌트를 완성했다고 보지 않는다.

```
Default   기본 상태
Hover     마우스 오버
Focus     키보드 포커스 (Tab)
Active    클릭/탭 누르는 순간
Disabled  비활성 (클릭 불가)
Loading   비동기 처리 중
Error     유효성 실패 또는 서버 오류
Success   완료 상태 (필요한 경우)
Empty     데이터 없음
```

## 18. 하지 말 것 (Anti-patterns)

**색**
- 그라디언트 배경 남용 (버튼, 히어로 배경 전체)
- 텍스트에 `text-shadow` 사용
- 색으로만 상태 구분 (색맹 접근성 위반)
- Primary 버튼 2개 이상 동시 노출
- neutral-400보다 밝은 색을 흰 배경에 텍스트로 사용

**타이포그래피**
- 같은 페이지에 3가지 이상 폰트 웨이트 동시 사용
- `font-size` 임의 값 (`text-[13px]` 등)
- 대문자 강조 남용 (heading 전체 uppercase)

**레이아웃**
- `100vh`를 height에 직접 사용 (모바일 주소창 이슈) → `dvh` 또는 min-h-screen
- 테두리 없이 흰 카드를 흰 배경 위에 올리기
- 같은 컨테이너 내 radius가 다른 요소 혼용

**인터랙션**
- `outline: none` 단독 사용 (키보드 접근성 파괴)
- 애니메이션 duration 600ms 초과
- z-index 임의 값 사용 (아래 테이블 외 값 금지)
- 모달 안 모달
- hover에 색이 2가지 이상 동시에 바뀌는 과한 피드백

**콘텐츠**
- 에러 메시지에 기술적 코드/용어 노출
- "확인", "예", "아니오"만 있는 모달 버튼
- 로딩 중 버튼을 비활성화하지 않아 중복 제출 허용

## 19. z-index 레이어 테이블

```
레이어          값      용도
─────────────────────────────────────────────────
base            0       일반 요소
raised          10      sticky header, floating 버튼
dropdown        20      드롭다운, 팝오버, 툴팁, 컨텍스트 메뉴
modal-backdrop  30      모달 오버레이 배경
modal           31      모달 컨테이너 (backdrop + 1)
toast           40      토스트 알림
critical        50      세션 만료, 최우선 경고
```

이 범위를 벗어나야 할 것 같으면 레이어 구조 자체를 재검토한다.
`z-[999]` 같은 임의 값은 절대 사용하지 않는다.

## 20. 개발 적용 체크리스트

컴포넌트 완성 전 아래 항목을 모두 확인한다.

**시각**
- [ ] 모든 색이 토큰(Tailwind 클래스)으로만 표현되어 있는가
- [ ] 임의 크기(`text-[13px]`, `w-[123px]`)가 없는가
- [ ] hover / focus / active / disabled 상태가 모두 구현되어 있는가
- [ ] 로딩 상태에서 스켈레톤 또는 스피너가 표시되는가
- [ ] 빈 상태가 설계되어 있는가

**접근성**
- [ ] 모든 이미지에 alt 속성이 있는가
- [ ] 아이콘 전용 버튼에 aria-label이 있는가
- [ ] 모달에 role/aria-modal/aria-labelledby가 있는가
- [ ] 에러 메시지에 role="alert"가 있는가
- [ ] Tab 키로 모든 인터랙티브 요소에 접근 가능한가
- [ ] focus outline이 제거되지 않았는가

**반응형**
- [ ] 모바일(360px)에서 레이아웃이 깨지지 않는가
- [ ] 텍스트가 overflow 없이 잘 줄바꿈되는가
- [ ] 터치 영역이 최소 44×44px인가
