# 기술스택 및 데이터베이스 확정 (P1)

**작성 목적**: 2026-03-16 피드백 반영 — 데이터베이스·기술스택을 확정하여 구현 시 일관된 선택 기준을 둔다.

---

## 1. 데이터베이스 확정

### 1.1 확정 사항

| 항목 | 확정 내용 |
|------|-----------|
| **DB 종류** | **RDBMS (관계형 DB)** |
| **제품** | **PostgreSQL** |
| **버전** | 15 이상 (또는 호스팅 제공 버전) 권장 |

### 1.2 선택 이유

- **RDB 선택**: 강의·수강·진도·Q&A 등 **테이블 간 관계와 정합성**(유일 제약, FK)이 중요하므로 RDB가 적합하다. ERD(erd.md) 설계가 RDB 기준으로 되어 있음.
- **PostgreSQL 선택**:
  - 오픈소스, 라이선스 부담 없음.
  - JSON/JSONB 지원으로 나중에 일부 스키마 확장 시 유연함.
  - 트랜잭션·인덱스·풀텍스트 검색 등 P1 요구사항에 충분함.

### 1.3 적용 시 유의사항

- ERD의 테이블·컬럼·유일 제약·인덱스를 **PostgreSQL DDL 또는 마이그레이션**으로 구현한다.
- 마이그레이션 도구: 프로젝트에서 선택 (예: Node 시 node-pg-migrate, TypeORM migrations; Java 시 Flyway/Liquibase 등).
- MongoDB를 사용할 계획이면 `docs/p1/submit/erd.md` 5장(JSON 구조)를 참조하며, **현재 P1 확정은 PostgreSQL(RDB)** 이다.
