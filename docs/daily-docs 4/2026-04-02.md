## 오늘 한 것

**피드백 서비스 전면 개편**
- 이용권 결제(`/feedback/buy`)와 피드백 신청(`/feedback/new`)을 별도 페이지로 분리
- 플랜당 3회 이용권 패키지 구조로 변경 (문서 39,900 / 영상 59,900 / 심층 99,900)
- `useFeedbackTickets` 훅: localStorage 기반 이용권 잔여 횟수 관리 (구매 시 +3, 신청 시 -1)
- `useFeedbackSubmissions` 훅: 신청 내역 localStorage 저장·조회

**피드백 신청 내역 페이지 (`/feedback/history`)**
- 전체·접수완료·검토중·완료 상태 탭 필터
- 카드 클릭 시 상세 확장, 진행 단계 스텝퍼 표시
- 완료 건에 전문가 피드백 내용 표시

**강의 카테고리 2단계 필터**
- 1단계(면접 방식) → API `category` 쿼리 / 2단계(직무·분야) → 클라이언트 필터링
- `FilterSidebar`에 그룹별 토글 UI 추가
- IT개발 10개 세부 기술 영역 + 기타 직무 9개로 세분화

**네비게이션 개선**
- 피드백 드롭다운 버튼을 우측 영역(로그인 버튼 옆)으로 이동
- 보유 이용권 수 뱃지 표시, 하위 메뉴 4개 (홈·구매·신청·내역)

## 오류 / 어려웠던 점

- 이용권 구매와 신청을 하나의 페이지에 두면 UX가 복잡해짐 → 페이지를 분리하여 각각 독립적인 흐름으로 구성
- 2단계 필터에서 백엔드가 `category` 단일 필드만 지원 → 1단계만 API, 2단계는 클라이언트 필터로 처리
