# P1 기획서 - 면접 인강 플랫폼 (고객 기능 중심)

**제출 문서: 기획서**

---

## 1. P1 목표 및 범위

- **주제**: 면접(인터뷰 준비) 인강 플랫폼
- **범위**: 고객(학생) 입장에서의 최소 기능 구현
- **산출물**: 백엔드 API, 프론트엔드 화면, DB 설계 및 구현

---

## 2. 최소 요구사항

### 2.1 백엔드
- REST API 제공 (OpenAPI 스펙: `api.yml`)
- 강의 목록/상세, 수강신청, 내 수업 목록, 영상 재생 정보, Q&A CRUD
- DB 연동 (ERD 기준 테이블 구현)
- 인증은 P1에서 단순화 가능 (예: 사용자 ID만 사용)

### 2.2 프론트엔드
- 강의 검색·목록·상세, 수강신청(장바구니/결제 흐름), 내 수업 조회, 수업 듣기(영상 플레이어), 질문하기(Q&A) UI
- 반응형 또는 웹 기준 레이아웃

### 2.3 DB
- ERD에 정의된 엔티티 구현 (강의, 강의섹션, 강의영상, 수강신청, 사용자, Q&A 등)
- 마이그레이션 또는 DDL 관리 (MongoDB 사용 시 컬렉션·JSON 구조는 본 제출의 erd.md 5장 참조)

---

## 3. 고객(학생) 입장 기능 상세

### 3.1 검색

| 기능 | 설명 | 상세 요구사항 |
|------|------|----------------|
| **키워드 검색** | 제목·설명·강사명으로 강의 검색 | 검색어 입력 시 실시간/버튼 검색, 빈값 시 전체 목록 |
| **카테고리 필터** | 면접 유형별 필터 | 기술면접, 인성면접, PT면접, 영어면접, 직무별(IT/금융/마케팅 등) |
| **난이도 필터** | 초급/중급/고급 | 단일 또는 다중 선택 가능 |
| **가격대 필터** | 무료/유료, 가격 범위(슬라이더 또는 구간) | 최소~최대 가격 설정 |
| **정렬** | 인기순, 최신순, 가격 낮은순/높은순, 평점순 | 드롭다운 선택 후 목록 갱신 |
| **검색 결과 표시** | 카드/리스트 형태 | 썸네일, 제목, 강사명, 가격, 평점, 수강생 수(선택) |

**화면 요구사항**: 검색창, 필터 영역(카테고리, 난이도, 가격대) + 정렬, 결과 목록 페이징 또는 무한 스크롤

### 3.2 수강신청

| 기능 | 설명 | 상세 요구사항 |
|------|------|----------------|
| **강의 상세 조회** | 수강신청 전 강의 정보 확인 | 커리큘럼, 강의 소개, 강사 소개, 목차(섹션·영상 목록), 가격, 예상 수강 시간 |
| **장바구니 담기** | 나중에 결제하기 위해 담기 | 담기 버튼 클릭 시 장바구니에 추가, 중복 담기 방지 |
| **장바구니 조회** | 담은 강의 목록 확인 | 강의별 삭제, 선택 삭제, 전체 금액 표시 |
| **수강신청(결제)** | P1에서는 간이 결제 | "무료 수강" / "유료 결제" 버튼 → 결제 완료 시 내 강의에 즉시 반영 (실 결제는 P3) |
| **수강신청 완료** | 내 수업으로 이동 | 완료 메시지 후 "내 수업 보기" 링크 제공 |

**화면 요구사항**: 강의 상세(상단 정보 + 탭 소개/커리큘럼/Q&A), 장바구니 페이지, 결제 완료 페이지

### 3.3 내 수업 조회

| 기능 | 설명 | 상세 요구사항 |
|------|------|----------------|
| **수강 중인 강의 목록** | 내가 수강신청한 강의만 조회 | 카드/리스트, 썸네일, 제목, 강사, 진도율 |
| **진도율 표시** | 강의별 시청 진행률 | % 표시, 진행 바 |
| **수료 여부** | 전체 영상 시청 완료 시 수료 | "수료", "수강 중" 뱃지 |
| **이어보기** | 마지막 시청 위치로 바로 이동 | "이어보기" 버튼 → 해당 영상 재생 화면 |
| **필터/정렬** | 수강 중/수료 완료 필터, 최근 수강순 등 | 탭 또는 필터로 구분 |

### 3.4 수업 듣기

| 기능 | 설명 | 상세 요구사항 |
|------|------|----------------|
| **영상 재생** | 강의 영상 시청 | 재생/일시정지, 구간 반복, 전체 화면 |
| **재생 속도** | 0.5x, 1x, 1.25x, 1.5x, 2x 등 | 속도 선택 시 즉시 적용 |
| **진도 저장** | 시청한 위치 자동 저장 | N초 단위로 서버 저장, 이어보기 반영 |
| **목차(커리큘럼)** | 섹션·영상 목록 클릭으로 이동 | 클릭 시 해당 영상 재생, 현재 영상 하이라이트 |
| **완료 체크** | 영상 시청 완료 시 수료 반영 | 90% 이상 시청 시 "완료" 처리 |
| **자막(선택)** | 자막 on/off | P1 선택 사항 |

### 3.5 질문하기

| 기능 | 설명 | 상세 요구사항 |
|------|------|----------------|
| **Q&A 목록** | 강의별 질문 목록 조회 | 강의 상세 또는 수업 듣기 내 Q&A 탭 |
| **질문 작성** | 제목, 본문 입력 후 등록 | 제목(필수), 내용(필수), 비공개(선택) |
| **질문 상세** | 질문 내용 + 답변 목록 | 작성자, 작성일, 수정일, 답변 리스트 |
| **답변 보기** | 강사/수강생 답변 조회 | 답변자, 답변 내용, 작성일 |
| **내 질문 수정/삭제** | 작성한 질문만 수정·삭제 | 수정 시 제목/본문 편집, 삭제 시 확인 |

## 4. P1에서 다루지 않는 것 (P2·P3)

- 역할별 권한(관리자/강사/학생) 세분화 및 인증 강화 → P2
- 실 결제 연동, 영수증, 환불 → P3
- 모니터링, 로깅, 스케줄링, 배치, 알림 발송 → P3
- 코드 리팩토링 및 성능 최적화 → P3
